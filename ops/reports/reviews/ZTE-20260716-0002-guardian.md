# Guardian Review — ZTE-20260716-0002

## Scope

Adversarial review of Pauli Cloud 1.0.0 release-candidate changes in PR #4.

## Attack categories

- branch and worktree constraint bypass;
- secret exposure;
- destructive or irreversible command execution;
- approval replay and expiration;
- policy-file mutation;
- unsafe Claude Code hook semantics;
- partial or irreversible installation;
- post-install drift and uninstall data loss;
- invalid phase transitions and false completion;
- prompt tampering and unmeasured promotion;
- fleet path injection;
- audit-ledger corruption;
- API authentication, timing, error disclosure, and rate abuse;
- container privilege and missing runtime capabilities;
- supply-chain, release, and rollback gaps.

## Findings and disposition

### G-01 — Generated Claude hooks used non-blocking exit codes

- Severity: HIGH
- Risk: Claude Code treats non-2 hook failures as non-blocking, allowing prohibited tool calls.
- Resolution: generated PreToolUse and Stop guards now exit `2` for blocking decisions; black-box tests invoke the generated scripts with Claude-style JSON stdin.
- Status: FIXED

### G-02 — Hook commands used relative paths and string form

- Severity: HIGH
- Risk: hooks fail outside repository root or behave differently across shells.
- Resolution: exec-form `command` plus `args` uses `${CLAUDE_PROJECT_DIR}` and explicit timeouts.
- Status: FIXED

### G-03 — Policy files were writable by the same agent they govern

- Severity: HIGH
- Risk: agent disables guards or edits approvals.
- Resolution: Edit/Write hook covers protected Pauli Cloud files and requires an active consequential-action approval ID.
- Status: FIXED

### G-04 — Environment boolean acted as irreversible approval

- Severity: HIGH
- Risk: approval bypass through a process variable.
- Resolution: generated and CLI guards validate expiring IDs against the approval registry.
- Status: FIXED

### G-05 — Policy compile could partially mutate before a later conflict

- Severity: HIGH
- Risk: incomplete controls and misleading install state.
- Resolution: transactional compiler runs a full dry-run preflight and applies only a conflict-free plan.
- Status: FIXED

### G-06 — Uninstall could overwrite post-install user edits

- Severity: HIGH
- Risk: user data loss during rollback.
- Resolution: uninstall compares current files with manifest hashes and fails closed on drift unless force is explicit.
- Status: FIXED

### G-07 — Generated guard ignored customized protected branches

- Severity: MEDIUM
- Risk: organization release branches remain mutable.
- Resolution: transactional compiler binds guard output to `config.json` protected branches and updates manifest provenance; idempotence is tested.
- Status: FIXED

### G-08 — API token comparison triggered insecure-hashing analysis

- Severity: MEDIUM
- Risk: inappropriate credential comparison and CodeQL failure.
- Resolution: padded constant-time byte comparison without credential hashing.
- Status: FIXED

### G-09 — API disclosed internal error messages and lacked request limits

- Severity: MEDIUM
- Risk: internal-path leakage and trivial abuse.
- Resolution: opaque 500 responses, security headers, fixed-window rate limits, no-store responses.
- Status: FIXED

### G-10 — Audit IDs were weak and ledger corruption was silent

- Severity: MEDIUM
- Risk: duplicate evidence and hidden damaged state.
- Resolution: cryptographic UUID event IDs, duplicate/parsing checks, corrupt-line reporting, enterprise readiness failure.
- Status: FIXED

### G-11 — Fleet accepted arbitrary non-repository paths

- Severity: MEDIUM
- Risk: unauthorized host-path registration and misleading fleet state.
- Resolution: realpath resolution plus Git-root validation.
- Status: FIXED

### G-12 — Prompt promotion relabeled experiment paths instead of creating canonical artifacts

- Severity: MEDIUM
- Risk: provenance ambiguity and mutable canonical state.
- Resolution: verified experiment content is copied to immutable canonical storage with measured-run provenance.
- Status: FIXED

### G-13 — Secret scanner produced opaque false positives

- Severity: MEDIUM
- Risk: release gate becomes unreliable or is weakened globally.
- Resolution: recognized reference expressions, reviewed-line markers, scanner tests, and value-suppressed CI artifacts.
- Status: FIXED

### G-14 — Runtime container lacked Git/SSH transport

- Severity: HIGH
- Risk: core inspection and checkpoint features fail after deployment.
- Resolution: validation and runtime images include Git, SSH client, and CA certificates.
- Status: FIXED

### G-15 — Repository settings cannot be enforced through source files

- Severity: HIGH release blocker, external configuration
- Risk: direct main pushes or unreviewed production release.
- Resolution: issue #5 and `docs/runbooks/GITHUB_CONTROLS.md` define required branch rules, security settings, production reviewers, and secret placement.
- Status: OPEN HUMAN CONFIGURATION GATE
