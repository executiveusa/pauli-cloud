# Guardian Review — ZTE-20260716-0003

## Scope

Focused adversarial review of the Pauli Cloud 1.0.1 hotfix for:

1. uninstall atomicity and drift handling;
2. verification-evidence freshness and phase binding;
3. approval-scope enforcement in generated Claude Code hooks.

## Attack results

### Uninstall atomicity — PASS

- Full manifest drift and backup preflight occurs before restore/delete mutation.
- Drift returns failure with an empty change set.
- The install manifest remains intact when preflight fails.
- Managed paths and backup paths are constrained to the repository root.
- Unexpected mutation failures restore snapshotted pre-uninstall state.
- Regression coverage proves a drifted late manifest entry leaves every managed file unchanged.

### Phase-bound evidence — PASS

- Verification evidence includes a unique verification ID, bead ID, phase, phase-start timestamp, and verification timestamp.
- The current phase start is derived from the append-only audit ledger.
- Stale, mismatched, pre-phase, unbound, or malformed evidence fails closed.
- Both `phaseAdvance(... COMMIT)` and `evaluateGuard(... phase-commit)` use the same binding contract.
- Existing `verifyProject` and enterprise verification APIs now produce phase-bound evidence.

### Approval scope — PASS

- Generated hooks require the approval ID, type, active status, expiry, and matching scope.
- Exact file scope authorizes only the named protected path.
- `project` scope preserves intentional project-wide authorization.
- Wrong scope fails closed at the actual generated-hook process boundary.

## Regression and integration review
